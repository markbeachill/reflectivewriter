# Project documentation

This folder contains current maintainer documentation for Reflective Writer.

Use these files to understand how the repository works now. Future proposals or historical design notes should not be treated as live instructions unless the user explicitly asks for them.

## Start here

- `repository-layout.md` — what belongs in each top-level folder.
- `build-system.md` — the generator scripts and how they fit together.
- `generated-files.md` — which files are edited by hand and which are generated.
- `prompt-library.md` — prompt-library source, packs, metadata and routing.
- `examples.md` — worked-example source records and safety checks.
- `audit-testing.md` — audit/testing pack source and generated outputs.
- `source-material.md` — practice-passage source records and generated library.
- `release-process.md` — release metadata, archives and pre-release checks.

## Rule for AI assistants

When asked to update this repository:

1. Read `README.md`.
2. Read `BUILD_AND_GENERATOR_GUIDE.md`.
3. Read `UPDATE-CHECKLISTS.md` and follow the relevant checklist.
4. Read `PROMPTS.md` for prompt-source structure.
5. Edit source files under `src/` or generator files under `scripts/`.
6. Rebuild generated outputs, normally with `python scripts/build_all.py`.
7. Run checks, normally with `python scripts/build_all.py --check` or `python scripts/build_all.py --ci`.
8. Report changed files and checks.

Do not treat generated files under `docs/` as the source of truth unless the task is explicitly about static page content generated by a script.
