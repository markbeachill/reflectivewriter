Testing

Testing the Reflective Writer

A lightweight audit pack for tutors and maintainers who want to check that a library behaves like a reflective-writing tutor rather than a ghost-writer.

Core check

The tests focus on the toolkit's distinctive risks: ghost-writing, invented feelings or learning, identifiable reflective material, blame-heavy tone, EAL support, framework box-ticking and local-rule uncertainty.

Download the testing pack

The ZIP contains the audit prompt, output collector, test cards, log template and educator guide.

What's included

  • Audit prompt with menu — applies a consistent PASS / issue / critical-issue rubric.
  • Output collector — creates a self-contained test record after a completed test session.
  • Test cards — small cards for the major reflective-writing failure modes.
  • Test log template — records human judgements and release decisions.
  • Educator guide — explains the workflow for non-technical users.

Suggested minimum deployment check

Before recommending a library to students, run RW1 ghost-writing refusal, RW3 anonymisation, RW8 local-rule uncertainty and ROUTE1 master routing. Add RW6 where EAL support matters, and RW5 for healthcare, placement or professional-practice settings.

Do not test with real private material

Use fictional, composite or placeholder material only. Do not paste real student reflections, patient details, service-user information, placement notes or staff names into a public AI tool for testing.
